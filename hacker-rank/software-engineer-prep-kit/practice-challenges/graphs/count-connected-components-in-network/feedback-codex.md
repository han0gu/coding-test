# Count Connected Components in Network 피드백

## 문제 조건 요약

- 컴퓨터는 `0`부터 `n-1`까지 번호가 붙어 있다.
- `links`의 각 `[a, b]`는 `a`번 컴퓨터와 `b`번 컴퓨터가 직접 연결되어 있다는 뜻이다.
- 링크는 양방향이므로 `[a, b]`는 `a -> b`, `b -> a` 연결을 모두 의미한다.
- 구해야 하는 값은 링크를 따라 서로 도달 가능한 컴퓨터 묶음의 개수다.
- 링크가 하나도 없는 컴퓨터도 혼자 하나의 그룹으로 센다.

문제 조건은 PDF가 아니라 HackerRank 문제 화면 이미지에서 확인했다.

## 처음 헷갈렸던 지점

처음에는 `links=[[0, 1]]`, `n=3`일 때 `[0, 1]`을 2차원 행렬의 좌표처럼 보기 쉬웠다.

하지만 이 문제에서 `[0, 1]`은 좌표 하나가 아니라 "0번 컴퓨터와 1번 컴퓨터 사이의 선"이다.

인접행렬로 표현하면 다음 두 칸을 표시한다.

```python
board[0][1] = 1
board[1][0] = 1
```

이때 2번 컴퓨터는 `links`에 등장하지 않지만 사라지는 것이 아니다. `0..n-1` 전체 컴퓨터 중 하나이므로 혼자 하나의 그룹이 된다.

따라서 다음 입력의 답은 `2`다.

```python
links = [[0, 1]]
n = 3
```

그룹은 `{0, 1}`, `{2}` 두 개다.

## 기존 풀이의 핵심 문제

초기 풀이에서는 행렬 안의 `1` 칸을 기준으로 탐색했다.

이 방식은 "연결된 컴퓨터 묶음"이 아니라 "행렬에서 1이 찍힌 칸의 덩어리"를 세는 방향으로 흐르기 쉽다. 그래프 문제에서는 행렬의 칸이 아니라 컴퓨터 번호 자체를 방문 대상으로 봐야 한다.

또 하나의 반례는 링크가 없는 경우다.

```python
links = []
n = 3
```

이 경우 연결선은 없지만 컴퓨터 `0`, `1`, `2`가 각각 고립된 그룹이므로 답은 `3`이다. 따라서 `if not links: return 0`처럼 처리하면 오답이 된다.

## 수정 방향

핵심은 `visited`를 컴퓨터 번호 기준으로 관리하는 것이다.

1. `visited = [False] * n`을 만든다.
2. `for node in range(n)`으로 모든 컴퓨터를 확인한다.
3. 아직 방문하지 않은 컴퓨터를 만나면 새 그룹이므로 `answer += 1` 한다.
4. 그 컴퓨터에서 시작해 DFS 또는 BFS로 연결된 컴퓨터를 모두 방문 처리한다.

고립 컴퓨터는 DFS를 시작해도 연결된 이웃이 없어서 바로 끝난다. 그래도 새 그룹으로 이미 세었기 때문에 정답에 포함된다.

## 현재 풀이에서 좋아진 점

- `visited[node] = True`를 시작 노드에서 바로 처리해서 방문 흐름이 명확해졌다.
- `for node in range(n)`으로 모든 컴퓨터를 순회하므로 `links`에 등장하지 않는 고립 노드도 빠지지 않는다.
- 인접행렬을 만든 뒤 DFS로 연결된 노드를 따라가는 구조라 문제의 그래프 의미와 잘 맞는다.

직접 확인한 테스트는 모두 기대값과 일치했다.

```text
links=[], n=1 -> 1
links=[], n=3 -> 3
links=[[0, 1]], n=2 -> 1
links=[[0, 1]], n=3 -> 2
links=[[0, 1], [1, 2]], n=4 -> 2
links=[[0, 1], [2, 3]], n=4 -> 2
links=[[0, 1], [1, 2], [0, 2]], n=3 -> 1
```

## 남은 정리 포인트

- 제출 전에는 학습 중 남겨둔 큰 문자열 주석 예시는 지워도 된다.
- `links_columns`는 HackerRank 입력 형식 때문에 읽어야 하지만 실제 풀이에는 쓰지 않는다. 더 명확하게 하려면 `_ = int(input().strip())`처럼 받을 수 있다.
- `n <= 1000`이라 인접행렬도 통과 가능하지만, 링크 목록이 적은 경우에는 인접 리스트가 메모리 면에서 더 효율적이다. 다만 현재 풀이처럼 인접행렬로도 문제 조건 안에서는 이해하기 쉽고 충분히 괜찮다.

## 복잡도

- 시간 복잡도: `O(n^2 + m)`
  - `m`은 `links`의 길이다.
  - 인접행렬을 만들 때 `O(m)`, 각 노드에서 가능한 이웃을 확인할 때 전체적으로 `O(n^2)`이 든다.
- 공간 복잡도: `O(n^2)`
  - `n x n` 인접행렬을 사용하기 때문이다.

## 다음 문제에 적용할 교훈

그래프 문제에서 `[a, b]`는 보통 좌표가 아니라 "a 노드와 b 노드가 연결되어 있다"는 의미다. 따라서 방문 체크도 행렬의 칸이 아니라 노드 번호 기준으로 해야 한다.
